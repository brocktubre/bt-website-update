Code Description Examples
newcomer assignment request When the newcomer specifically asks to be assigned to an issue or if they ask if they could work on that specific issue. "Hi, I'm a newcomer to golang and I'd like to work on this issue. Can I be assigned with this issue?" "Can I send a PR for this issue?"
acceptance inquiry The newcomer is asking if he or she does XYZ, would the PR be accepted. "Do you have any comments on what changes would be most likely to be accepted? I am reluctant to do any work on this if it is going to be rejected or ignored."
issue status inquiry When someone wants to know if someone else is already working on an issue, or if no one is assigned to the issue. "@kkang2097 hey is it done?"
"Hey Guys, Is the work done?"
starting guidance request Someone seeking guidance about how to start, the contribution process, or high level help on how to proceed. "Hi @Griffintaur, This is a pretty cool game, I'd like to help contribute.Can you give me more context or where I can start with this? Thanks!"
"Hmm, okay. Is there any guides you can point me to for how to contribute to open source projects?"
"I want to work on this issue. Can you please guide me more about the same?"
coordination inquiry Asking when something needs to be done or if a specific date for finishing the work is okay. "Is next Sunday OK for me to finish?"
PR review request Asking for someone to review a pull request. @erezrokah Please review #3597.
code location inquiry Asking information about which artifact needs to be changed to fix the issue; asking if the file they are changing are the correct ones. "Hi @owickstrom, I want to start contributing to the repository beginning with this but the referenced file seems to be obsolete already. Can you identify where the function was moved? Thanks!"
validation request The newcomer shares a potential solution to the issue and seeks validation from the community. @raphaelm So as per my understanding, I should be using one script tag inside the index.html page with the JSON template and load variable data. Am I on the right track?
resource inquiry The newcomer seeks resources such as documentation or tools. I've contributed to repos before using the tooling available on the website, I just haven't used the git command-line.
The Contributing Guidelines page doesn't appear to highlight all the project dependencies. Is there any documentation on that?"

(This is different from PR review request because there is no PR in place, an the newcomer is asking if feedback to the code they are working on) @scrthq So just to confirm, I will add an entry there similar to json.

mentoring request Asking for general guidance from another person or contributor. I would like to take this up. This is going to be my first contribution. Is it still available? Also can someone guide me where can I start to work on this?
technical related question When someone is asking technical questions about the code or having issues with something specific about the project. I am getting this error when trying to call my yoloFormater.toYOLO() function defined inside my yolo.js file. Any idea what the issue is, and how I might solve it?

Code Description Examples
coordination statement Here the newcomer is mentioning dates, schedules, or planning task completion "I will work on that issue after work, in order to submit a PR + code review tomorrow morning."
knowledge share The newcomer shares information about either a library or a resource. They also might be answering a questions about why they did what they did. Here they are sharing new information with the project team. "I took a different approach and modified `LabelEncoder` to track the class labels and wrote a wrapper `Estimator` class that encapsulates calling `transform` the labels in `fit` and `inverse_transform` in `predict`. It works on all estimators and is much easier than modifying them all separately--all it does is map integers to classes and maps them back later. (Also I have an `Ensemble` class that uses `Estimator`s and has a voting or averaging function on calls to `predict`.)"
suggestion When someone suggests to change or do something differently. Suggestion: rather than turning setters into public, perhaps have a factory method that forces to pass in all the parameters to initialize a new `EventData`? Same with `SystemPropertiesCollection`,
Maybe this approach would work out for scikit. It solves all the issues I had with class labels / ensembles and it very little code.
grateful Saying "thank you", grateful, saying something like "that was helpful". @faho, @pickfire: Thank you both for your comments.
acknowledgement Saying something like "okay" or "that makes sense" "Ok, I understand."
general statement The newcomer is just saying something. "I was browsing through previous year's GSOC projects and happened to come across this !. Also, a friend of mine (who also contributes to this project) told me about this."
